УВАГА: якщо я не указую типи масивів, значить вибирайте їх самі.

1. Створіть 2-вимірний масив любого розміру і ініціалізуйте його при створенні. Виведіть всі елементи на екран так, 
   як вам подобається.

2. (Доповнення до задачі 1). Знайдіть суму всіх елементів в масиві і виведіть її.

3. Створіть два масиви розміром 2 на 3 і ініціалізуйте їх при створенні. Створіть третій масив 2 на 3, який буде сумою
   двох масивів. Тобто в ньому кожний елемент - це сума відповідних елементів двох інших масивів. Виведіть всі 3 масиви
   на екран в прямокутній формі, щоб можна було перевірити чи правильний код ви написали.

4. Створіть масив 3 на 4, заповніть його. Найдіть в ньому найбільше і найменше значення. Виведіть їх на екран, після
   чого поміняйте їх місцями в масиві. Виведіть весь масив на екран до перестановки і після неї, щоб перевірити, чи
   дійсно значення змінилися.

5. Створіть масив 3 на 3. Перевірте чи він симетричний. Лінія, відносно якої перевіряється симетрія - діагональ.

6. Попросіть у користувача ввести розміри масиву. Після цього створіть масив відповідних розмірів і дайте користувачу
   ввести значення кожного його елементу. Після цього виведіть масив на екран.

7. Створіть масив 3 на 4. Виведіть на екран суму кожного окремого стовпчика. А потім виведіть суму кожного окремого рядка.

8. Створіть масив 3 на 3, який буде слугувати доскою для хрестиків-ноликів. Спочатку всі його елементи заповнюються символом
   ' ' (пробіл). Далі по черзі питайте в користувачів, куди вони хочуть поставити нулик/хрестик. Нехай спочатку ходить
   гравець, який ставить нулики. Попросіть у нього ввести два індекси (номер стовпчика і рядка). Перевірте чи він вводить
   правильні дані. Якщо номери вилазять за межі масиву - так і кажіть. Якщо це місце уже зайнято - так і кажіть. Після
   кожної помилки дайте користувачу знову ввести символ. Після цього давайте другому користувачу ввести хрестик. І ця дія
   виконується до того моменту, поки хтось не виграє, або поки не буде нічия. Тобто після кожного вводу символу ви маєте
   перевіряти вашу доску. Результат гри також виводьте гравцям на екран.

9. Є масив 3 на 3 із букв. В ньому закодоване повідомлення. "It is key" (рівно 9 символів, по 3 символи на рядок).
   Але кожна буква посунута на 5 символів вправо. Тобто ініціалізація масиву наступна:
   
   int encryptionKey = 5;
   int[,] message =
   {
      { 'I' + encryptionKey, 't' + encryptionKey, ' ' + encryptionKey },
      { 'i' + encryptionKey, 's' + encryptionKey, ' ' + encryptionKey },
      { 'k' + encryptionKey, 'e' + encryptionKey, 'y' + encryptionKey }
   };

   Виведіть на екран всі ці елементи в один рядок (буде незрозумілий набір символів), щоб користувач побачив закодоване
   повідомлення. Попросіть у нього ввести ключ для розкодування повідомлення. Це має бути значення 5. Якщо він введе
   5 - відніміть від кожного елементу значення 5, покажіть правильне повідомлення і привітайте. Інакше покажіть
   некоректне повідомлення (там буде незрозумілий набір символів) і скажіть що ключ не вірний. Замітьте, що масив
   має тип int, а вам треба буде виводити кожний елемент на екран, як символ, а не як число.
